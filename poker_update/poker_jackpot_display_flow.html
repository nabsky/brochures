Poker Jackpot Display
Screen Flow

Working overview of the main table display states, dealer interaction flow, jackpot confirmation sequence, and network announcement behavior.

Main operational flow

  1. Table starts offline and waits for backend connection.
  2. Once connected, jackpots are visible even before dealer login.
  3. Dealer logs in and opens a new session.
  4. During the betting phase, the dealer enters the number of bonus bets.
  5. The same number of chips flies onto the screen as a visual representation of bonus entries.
  6. Dealer confirms with Enter; chips fly upward and progressive jackpot values increase.
  7. Local table jackpots are selected and confirmed directly on the table.
  8. Progressive jackpots require jackpot selection plus PIN confirmation.
  9. After confirmation, a network-wide progressive jackpot announcement screen may be shown across all tables.

Operational Flow Overview

The display is built around a small number of operational states: connection state, dealer session state, active jackpot betting state, local jackpot confirmation, and progressive jackpot authorization. The goal is to keep the live table UI readable while still providing strong visual emphasis for important jackpot events.

1
Connection state. The table first comes online. Before that, the UI stays in offline mode and clearly indicates that the system is connecting to the server.
2
Dealer session start. After the table is online, jackpots can already be displayed. The dealer then logs in and confirms the start of a new session.
3
Accepting jackpot bets. During the active betting phase, the table displays progressive and local jackpots as the main live values.
4
Entering bonus bets. The dealer presses the number of bonus bets placed. The same number of chips appears on the screen as a visual confirmation of pending entries.
5
Committing entries. The dealer confirms with Enter. The chips fly upward, and the progressive jackpot amounts increase accordingly.
6
Local jackpot flow. If a local table jackpot is hit, the dealer selects the winning combination, then confirms the payout on the same table.
7
Progressive jackpot flow. If a progressive jackpot is hit, the dealer selects the jackpot and the authorized person completes PIN confirmation before the result is finalized.
8
Announcement state. After progressive confirmation, a dedicated premium announcement screen can be broadcast to all tables in the network.
Dealer-driven actions
Login confirmation, bonus bet count entry, local jackpot selection, progressive jackpot selection, and local payout confirmation.
Authorized actions
PIN confirmation for progressive jackpots, used to separate selection from final approval and reduce accidental confirmations.
Animated feedback
Chips appear according to the entered bonus bet count, then fly upward on confirmation. Progressive cards use stronger highlight treatment than local cards.
Network output
Progressive wins can trigger a premium full-screen announcement shown across the full table network.
01

Offline / No Active Connection

Base state shown before the table has connected to the backend server. The header shows OFFLINE, and the main card communicates that the table is closed while the system is connecting.

Purpose: clearly show that the table is unavailable and cannot accept gameplay actions.
02

Online / Dealer Not Logged In

The table is connected and jackpots are already visible, but there is no active dealer session yet. The main state still remains TABLE CLOSED until a dealer logs in and opens the table.

Purpose: expose live jackpot values while still making it clear that the table is not open for play yet.
03

Dealer Login Confirmation

Transitional welcome screen shown after dealer identification. It confirms the dealer session and can optionally display details about the previous session for quick context.

Purpose: visually transition the table from closed state into an active dealer session.
04

Accepting Bets

Main live screen during the jackpot betting phase. The header shows the current jackpot bet amount, and both progressive and local jackpots are displayed in their regular active state.

Purpose: serve as the primary working screen while jackpot bets are being accepted.
05

Bonus Bets Entered / Chips Appear

The dealer enters the number of placed bonus bets. The same number of chips flies onto the screen, giving a direct visual representation of pending bonus entries before they are committed.

Dealer interaction: the dealer presses the number of placed bonus bets. The same number of chips appears on screen. After pressing Enter, the chips fly upward and the progressive jackpot values increase.
06

Table Jackpot Selected

When a local fixed jackpot is hit, the dealer selects the winning hand. The selected card receives a stronger border and brighter state, but the result is not finalized yet.

Purpose: show the dealer’s current choice before confirming the local jackpot payout.
07

Table Jackpot Confirmed

After confirmation, the local jackpot area switches into a dedicated confirmed-win state with hand name, payout amount, and a clear instruction for closing the screen.

Purpose: clearly lock and display the confirmed local jackpot result on the current table.
08

Progressive Jackpot Selected

When a progressive jackpot is selected, its card receives stronger emphasis than a local jackpot card: brighter gold border, stronger focus treatment, and elevated visual priority.

Purpose: show which progressive jackpot has been selected before the authorization step.
09

Progressive Jackpot PIN Confirmation

Technical confirmation screen for a progressive jackpot. The jackpot type and amount are already selected, and an authorized person enters a PIN code to finalize the event.

Purpose: separate jackpot selection from final authorization and reduce accidental confirmations.
10

Progressive Jackpot Confirmed

After successful PIN confirmation, the progressive jackpot is locked in as confirmed. The main layout returns, while the winning progressive card remains visibly highlighted on the table display.

Purpose: show that the progressive jackpot has been successfully confirmed and completed on the current table.
